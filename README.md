# GeoAware

GeoAware is a personal project designed to help manage the large volume of emails generated by my activity in the geocaching community.

⚠️ **Important:** This is a personal tool created primarily to solve my own workflow problems. The initial goal is simply to make things work and save time. Code quality, architecture, and efficiency are secondary concerns during early development. Improvements and refactoring may happen later.

---

# About Geocaching

[Geocaching](https://www.geocaching.com/) is a real-world outdoor treasure hunting game where players use GPS-enabled devices to find hidden containers ("geocaches") placed by other players.

Each cache typically includes:
- A hidden physical container
- A logbook to sign
- An online listing describing the location and puzzle (if any)

Players record their visits online, write logs, discuss caches, and sometimes exchange messages with cache owners.

Geocaching has been my hobby for more than **10 years**.

---

# The Problem

Over the years my level of activity in the geocaching community has increased significantly:

- I follow many caches (watchlist notifications)
- I own and maintain **200+ caches**
- I actively participate in feature discussions and feedback forums
- I own **53 EarthCaches**, which require reviewing answers from visitors
- I receive logs, questions, and messages in multiple languages

Because of this, my email inbox receives **dozens to hundreds of messages per day**.

These emails include:
- cache logs
- private messages
- watchlist notifications
- forum discussions
- new cache publications
- EarthCache answers that require validation

Sorting, reading, and responding to all of this manually takes a significant amount of time.

---

# Why GeoAware Exists

The official geocaching platforms do not allow automation of their internal systems. However, most notifications are delivered via email.

GeoAware therefore focuses on **automating the management and interpretation of geocaching-related emails** using the **Google Gmail API**.

The goal is not to automate geocaching itself, but to **reduce the manual effort required to process large volumes of email notifications.**

---

# Planned Features

## Email Classification

Automatically classify incoming emails into structured categories such as:

- Messages (private message center)
- Cache logs
- New cache notifications
- Watchlist activity
- Forum discussions

Although Gmail provides filtering, the variability of geocaching email formats often makes built-in filters unreliable.

GeoAware aims to provide more accurate classification.

---

## EarthCache Identification

Detect messages related to my **EarthCaches**.

This requires maintaining a **static database** containing:

- GC codes
- cache names
- question sets
- expected answers

This database will initially contain my **53 EarthCaches**.

---

## Automatic Answer Scoring

When a geocacher sends answers for an EarthCache:

1. The system extracts the answers
2. Compares them with the expected answers
3. Generates a **score**

Based on this score, the system can automatically generate a response message.

Since the **geocaching message center accepts email replies**, it is possible to respond automatically by replying to the received email.

---

## Rule Compliance Detection

According to EarthCache guidelines, visitors must **send their answers before logging the cache**.

GeoAware should detect situations where:

- A user logs the cache without sending answers
- Answers are sent **after** the log
- Multiple people from the same group send answers separately

Handling these edge cases is part of the system design.

---

## Daily Log Summaries

Generate daily summaries for:

### My caches
A digest of all logs received that day.

In many cases logs contain little content, so this helps save time.

### Watched caches
A daily summary of logs for caches I follow, mostly for curiosity.

---

## EarthCache Answer Storage

Store all received EarthCache answers in a database.

This allows:

- tracking previous submissions
- detecting copied answers
- identifying commonly shared incorrect responses

Incorrect answers are unfortunately very common in EarthCaches.

---

# Technical Approach

GeoAware will rely primarily on:

- **Google Gmail API** for email access
- Email parsing and classification
- Local data storage for EarthCache metadata and submissions

The system focuses on **email-based automation**, avoiding any interaction with geocaching.com internal systems.
